存储引擎的概念

  • 用来处理数据库的相关 CRUD 操作
  • 每个数据库都有存储引擎的概念
  • Oracle 是不特别强调存储引擎这个概念的 Mysql 同样
  • Oracle
    • 堆表
    • 簇表
    • 索引组织表

Mysql存储引擎

  • 官方存储引擎

    • MyISAM

      • MySQL5.1版本之前的默认存储引擎
      • 堆表数据结构
      • 表锁设计
      • 支持数据静态压缩
      • 不支持事务
      • 数据容易丢失
      • 索引容易损坏
      • 相比起来唯一的优点
        • 数据文件可以直接拷贝到另外一台服务器使用
      • MyISAM 数据文件
        • .MYD
      • 索引文件
        • .MYI
      • 修复错误数据表
        • myisamchk

    • innoDB

    • Memory

      • 全内存存储的引擎
      • 数据库重启后数据丢失
      • 支持哈希索引
      • 不支持事务
      • Memory 存储引擎是不支持禁用的
      • MySQL 内部的排序操作的临时表使用该存储引擎
      • 参数 max_heap_table_size 决定使用的大小
        • 默认的大小：16M
      • 内存放不下的转为MyISAM表
        • 注意临时路径空间大小
      • 内除使用为会话级别
        • 小心 OOM

    • Federated

      • 运行本地访问远程 MySQL 数据库中表的数据
      • 本地不存储任何数据文件
      • 类似 Oracle 中的透明网关
        • 仅支持 Mysql => MySQL 的访问
        • 不支持异构数据库的访问
      • Federrated 存储引擎默认不开启
        • 配置文件：federated
      • 异构数据库远程访问
        • MariaDB ：FederatedX 支持 ODBC 的方式访问异构数据库

    • CSV

      • Comma Separated Values
      • CSV 是一种标准文件格式
      • 文件以纯文本形式存储表格数据
      • 不支持特殊字符
      • 使用相当广泛
        • mysql 数据库中 slow_log 表和 general_log 表使用的就是用的CSV存储引擎

    • Archive

      • 是无法在 OLTP 类的应用中所使用的，因为它不支持对数据的修改，而OLTP中一定会对数据进行修改，对于一些数据仓库的应用，或者一些特殊的表，如：记录日志类的表或者数据采集的表，由于会要存储大量的数据，就比较适合 Archive 存储引擎，存储空间占用最小。

  • 第三方存储引擎

    • TokuDB
      • 插入密集型可以用
    • Spider
      • 分布式中间件
